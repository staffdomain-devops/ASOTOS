| Make.com → GitHub API | External HTTP caller sends contact_id and contact_email as workflow inputs |

| GitHub Secrets → Python env | Four API keys enter Python via job-level env block |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | All stub scripts | mitigate | Never log CONTACT_EMAIL in print statements; log only CONTACT_ID. Never call print(os.environ) or run: env in steps. |
| T-01-02 | Tampering | requirements.txt | mitigate | Exact == pins on all 9 packages prevents silent upgrades on GitHub-hosted runners. Verified versions from PyPI. |
| T-01-03 | Tampering | workflow_dispatch inputs | accept | contact_id and contact_email are routed only to env vars, never interpolated into shell commands (no "run: echo $CONTACT_ID"). Python os.environ.get() is safe from shell injection. |
| T-01-04 | Information Disclosure | GitHub Actions run logs | mitigate | GitHub masks exact secret strings; stubs do not log derived or partial secret values. contact_email is an input not a secret — only log contact_id. |
</threat_model>
